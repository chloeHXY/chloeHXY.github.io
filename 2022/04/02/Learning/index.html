人脸识别

机器学习的人脸识别过程

基于传统图像处理和机器学习技术的人脸识别技术，其中的流程都是一样的。

机器学习-人脸识别系统都包括：

  • 人脸检测（解决人在哪的问题）

  • 人脸对齐（通过找基准点，让人脸图像都变换到一个统一的角度和姿态）

  • 人脸特征向量化（特征向量用于描述这张人脸。）

    所以我们常常利用PCA技术对人脸描述向量进行降维处理，保留数据集中对方差贡献最大的人脸特征来达到简化数据集的目的

  • 人脸识别（对上一步人脸向量进行分类，使用各种分类算法，以达到人脸识别的目的）

深度学习的人脸识别过程

不同于机器学习模型的人脸识别，深度学习将人脸特征向量化，以及人脸向量分类结合到了一起，通过神经网络算法一步到位。

二、计算机网络

SSH

（secure shell 安全壳） 安全网络协议 专为远程登录服务会话、其他网络服务提供安全性的协议。

可以加密传输数据（1） 压缩数据 提高传输速度

/Windows中使用SSH，需要使用另一个软件PuTTY。/

（1）安全机制

RSA非对称加密技术

（2）安全验证方式

ssh有两种级别的安全验证方式：

1.口令登录

即需要输入密码登录到远程主机

2.使用密钥免密登录

大概是，在客户端生成密钥对，将公钥拷贝到远程主机，当客户端发送请求时，会在服务器中寻找该客户端的公钥，并与传来的公钥对比，相同则发送质询到客户端，用私钥解密后，传递登录信息到服务器。

对于口令登录，虽然在传输过程中使用了rsa进行加密，但可能存在冒充服务器的情况，即中间人攻击，

而使用密钥登录，无需传输口令，避免了中间人攻击。

FTP(File Transfer Protocol)

文件传输协议，它是TCP/IP协议组中的协议之一

包括两个组成部分：1.FTP服务器 2.FTP客户端

利用FTP协议将网页、程序传到web服务器上

？？？用户使用FTP客户端通过FTP协议访问FTP服务器上的资源

你还需要一种将文件从本地硬盘上传到远程Web服务器的方法。 为了做到这一点，你必须使用FTP客户端。

每个系统都包含一个FTP客户端作为文件管理器的一部分。 Windows资源管理器，Nautilus（一个常见的Linux文件管理器）和Mac Finder都包含该功能。 然而，人们更经常选择专门的FTP客户端，可以存储密码并且并行显示本地和远程目录。

网络服务器

硬件上：计算机

与其他连接到互联网的设备进行物理数据的交互

软件上：HTTP服务器

一台HTTP服务器是一种能够理解URL（网络地址）和HTTP（浏览器用来查看网页的协议）的软件

HTTP（超文本传输协议）

  • 只有用户可以制定 HTTP 请求，然后只会发送到服务器。服务器只能响应客户端的HTTP请求。
  • 当通过 HTTP 请求一个文件时，客户必须提供这个文件的URL。
  • 网络服务器必须应答每一个 HTTP 请求，至少也要回复一个错误信息。

TCP

传输控制协议，它使两台主机能够建立连接并交换数据流。TCP 能保证数据的交付，维持数据包的发送顺序。确保数据传输的可靠性

客户端

展现一个网页的过程

浏览器发送请求（获取页面的HTML文档），解析文档资源（如：css，图片视频等页面资源），继续发送请求，获取可执行脚本、css样式进行页面渲染，最终呈现一个完整的网页。浏览器获取到的可执行脚本可以在之后的阶段获取更多资源，并相应地更新网页。

总之：浏览器负责发送HTTP请求，并解析HTTP返回的响应，以向用户提供明确的响应。

HTTP本质是无状态的，使用Cookies（网页浏览数据）可以创建一个会话让每次请求都能共享相同的上下文信息，达成相同的状态。

HTTP缓存

缓存是一种保存资源副本并在下次请求时直接使用该副本的技术。复用了以前的资源，大大提升了性能。

DNS（域名服务器）（Domain Name Server）

IP地址和域名的对应关系存放在域名服务器中，其可提供IP地址和域名之间的转换服务

IP地址与域名

IP地址就像人的身份证号：127.0.0.1(就是localhost)

域名就像人的姓名：www.baidu.com

端口号

只有80端口可以被省略，非80端口，都需要 localhost:xxxx

四、OS(计算机操作系统)

【OS】DOS

是什么？

DOS(Disk Operation System)磁盘操作系统，通过dos语言（命令）对电脑系统中的磁盘文件进行操作管理。

在哪里？

windows操作系统下通过cmd即可运行dos环境

同为操作系统，和windows有何异同？

Dos实际上先于windows出现，最大的不同是windows相较于dos实现了图形化界面，无需输入命令，鼠标点击即可，且我们一次可以打开多个程序窗口，而不像dos需要记忆大量命令，还是单任务单对象。

但dos仍然有很多优点，比如一些远程操作等高级操作，dos只需要一行命令即可实现，而windows却要一直点点点/(ㄒoㄒ)/~~

而且dos的速度非常快，效率非常高。

【OS】关于\r 、\r\n 、 \n的区别

\n：换行 （换到当前位置的下一行，而不是行首）

\r：回车（回到当前行的行首，可能会覆盖该行）

在windows系统中，实现我们所想的”换行”实际是回车换行，需要用\r\n实现，而linux系统下则只需要\n

但我们编写某些高级语言程序时，发现，使用\n也可以达到我们想要的效果，这是因为，编译器，帮我们将\n自动转换为了\r\n

进程：拥有资源的最小实体，（一个进程中可能有许多线程在使用其资源）

☀进程是程序在一个数据集合上的运行过程，是系统进行资源分配和调度的一个独立单位

node.js的模块化概念

CommonJs是模块化的标准。广泛应用于Nodejs、vue、react等框架语言

模块是Node.js应用程序的基本组成部分，文件与模块是一一对应的。一个Node.js文件就是一个模块。

可以认为每个模块都是一个module对象，包含两个对象 ：require（从外部获取一个模块的公开接口） exports（模块公开的接口）

想让外部可以访问模块里面的方法或属性，就必须通过exports/module.exports暴露属性或方法

1.将想让外部模块使用的方法/属性添加为exports对象的成员变量

2.也可以直接将module.exports=一个对象，直接暴露一个对象

最好不要同时使用上述方法，可能存在覆盖。

可以认为var hello=require(‘./hello’)； hello就是hello.js中的exports对象

请求的内容又叫请求报文，相应的还有响应报文

以下内容，在调试时，都可以通过浏览器控制台查看，是🚩后为具体位置

☀请求报文的格式：

行： METHOD(如POST、GET、DELETE……) URL HTTP协议的版本号

头：pg: HOST: xxx

​ xxx: xxx 🔔：格式均按照名称+冒号+空格+值 🚩 请求行、头都在RequestHeaders

空行

体：（若是get请求，则请求体为空） 🚩post请求在：FromData get请求可在 QueryStringParameters中查看查询字符串参数

☀响应报文的格式：

行：HTTP协议版本号 响应状态码（如200，404，403） 状态字（其中200：OK 404：NOT FOUND）

头：格式同上 🚩 请求行、头都在ResponseHeaders

空行

体：响应体是html语言 🚩Response

【AJAX】关于IE缓存

在使用IE浏览器作为客户端发送AJAX请求时，存在响应缓存问题

即若两次请求的method,url地址相同，则直接从缓存区获得上一次的响应结果。

解决方法：（针对get方法）添加查询字符串参数 t=Data.now() 使每次请求的url地址都不同。
